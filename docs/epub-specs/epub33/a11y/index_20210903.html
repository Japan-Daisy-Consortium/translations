1. 導入

1.1 概要

このセクションは規定ではない。

この仕様、EPUB Accessibility は、EPUB® エコシステムにおける二つの重要なニーズに対応している。

  • EPUB 出版物のアクセシブルな品質の発見可能性、及び
  • アクセシブルなEPUB 出版物の評価と認証。

アクセシビリティ メタデータの提供は、EPUB 出版物のユーザビリティに関する情報に基づいた意思決定を容易にする。消費者は、コンテンツの品質を評価し、EPUB 出版物がアクセシブな保証の基準を満たしているかどうかに関わらず、自分固有のニーズに適しているかどうかを判断することができる。最低限、この仕様書に適合する全ての EPUB 出版物は、§ 2. 発見可能性に記載されているアクセシビリティ メタデータ要求事項を満たしている。

これまでも EPUB 制作者は、高度なアクセシビリティを備えた EPUB 出版物を制作できたが、この仕様書は、コンテンツがアクセシブルであると認証するための正式な要求事項を定めている。これらの要求事項は、そのコンテンツを評価するための明確なガイドラインのセットを EPUB 制作者に提供し、品質の保証を可能にする。アクセシブルな EPUB 出版物は、§ 3. アクセシブルな出版物で説明されているアクセシビリティ要求事項を満たしているものである。

仕様書は、EPUB 制作者が、特定のユーザーのニーズに合わせて最適化したために、幅広いアクセシビリティ要求事項を満たすことができないコンテンツを識別する方法も確立している。詳細は、§ 4. 最適化された出版物の最適化された出版物の要求事項を参照されたい。

この仕様書は、EPUB の単一のバージョンを対象としない。規格の将来のバージョンを含む、任意のバージョン又はプロファイルに適合する EPUB 出版物に適用可能である。

理想的には、これらのガイドラインは、オープン ウェブ テクノロジー上に構築された全てのデジタル出版物の評価に役立つことだが、そのような適用を保証することは、この仕様の範囲外である。

注

この仕様書に至った決定の背景については、EPUB Accessibility Frequently Asked Questions [AccessibilityFAQ] を参照されたい。

1.2 達成方法集

このセクションは規定ではない。

この仕様書は、WCAG [WCAG2] が、それらを達成するための達成方法集からアクセシビリティ ガイドラインを分離した方法と同様に、EPUB 出版物のアクセシビリティ要求事項として抽象的なアプローチを採用している。このアプローチは、フォーマットが進化しても安定した状態を保つガイドラインとなる。

このアプローチを容易にするために、手引書の EPUB アクセシビリティ達成方法集 [EPUB-A11Y-TECH-11] ドキュメントは、適合性の達成方法集を概説する。これらの達成方法集は、EPUB の異なるバージョンに対して、この仕様書の要求事項を満たす方法を説明する。

1.3 仕様への適用

このセクションは規定ではない。

この仕様書は、（例えば、EPUB 2 [OPF-201] など）この仕様書を参照していない旧バージョンのEPUBに適合したコンテンツであっても、任意の EPUB 出版物に適用可能である。

このような EPUB 出版物の制作者は、この仕様書のアクセシビリティと発見可能性の要求事項に適合したコンテンツを制作するべきである。EPUB 制作者もまた、最先端のアクセシビリティ機能や達成方法集にアクセスするために、EPUB の最新版にアップグレードするべきである。

1.4 用語

この仕様書は、EPUB 3 [EPUB-3] で定義されている以下の用語を使用する。

  • EPUB コンテンツ ドキュメント
  • EPUB 制作者
  • EPUB 出版物
  • メディア オーバーレイ ドキュメント
  • パッケージ ドキュメント
  • リーディング システム

加えて、[WCAG2] で定義されている支援技術の定義を使用する。

注

支援技術は、必ずしもリーディング システムとは別個のアプリケーションではない。リーディング システムは、例えば、音声合成再生など、しばしば、単独で動作する支援技術の機能を統合している。

1.5 適合性

この仕様書にある全てのオーサリング ガイドライン、図、例、及び注釈は、規定ではないとマークされたセクションと同様に、規定ではない。この仕様書の他の全ての項目は、規定である。

このドキュメントのキーワード MAY、MUST、MUST NOT、OPTIONAL、RECOMMENDED、SHOULD は、次に示すように、全て大文字で表示されている場合に限り、BCP 14 [RFC2119] [RFC8174]に記述されているように解釈される。

2. 発見可能性

2.1 導入

このセクションは規定ではない。

ウェブ ページとは異なり、EPUB 制作者は、個人消費のために多くのチャンネルを経由して EPUB 出版物を配信する。EPUB は、電子書籍及び他のデジタル出版物の種類において成功したフォーマットを作ったモデル。しかしながら、このモデルの結果は、出版物のアクセシビリティに関する特定の詳細が、それと一緒に伝えなければならない。

例えば、発行者と製作者からコンテンツを集めているオンライン書店は、発行者がメタデータを介して通知しない限り、それぞれの提出物に費やされた制作品質を知ることができない。

全ての利害関係者が、EPUB 出版物のアクセシブルな品質を発見できることを保証することは、第一の関心事である。ユーザーが、EPUB 出版物を購入したり、借りたり、別な方法で入手するとき、EPUB 出版物のユーザビリティを評価できなければならず、これにはアクセシビリティ要求事項を満たすために作られたアフォーダンスを知る必要がある。

同様に、この仕様書のアクセシビリティ要求事項を満たしていないコンテンツでも、個々のユーザーのニーズを満たすことができないとは限らない。

豊富なメタデータの提供を通してのみ、ユーザーは、それらのコンテンツが自分に適しているかどうかを判断できる。

2.2 パッケージ メタデータ

全ての EPUB 出版物は、出版物がアクセシビリティ又は最適化の要求事項を満たしているかどうかに関わらず、そのアクセシブルなプロパティを公開するパッケージ ドキュメントにアクセシビリティ メタデータを格納しなければならない（MUST）。

EPUB 出版物には、次のアクセシビリティ メタデータを格納しなければならない（MUST）。

  •  アクセス モード — （例えば、テキスト、視覚、聴覚、触覚など）コンテンツを処理又は理解するために必要な人間の知覚システムや認識能力。

  •  アクセシビリティの機能 — （例えば、代替テキスト、拡張した概要、キャプションなど）コンテンツの全体的なアクセシビリティに貢献する機能や適合。

  •  アクセシビリティの危険性 — （例えば、発光、動作のシミュレーション、サウンドなど）コンテンツが提示する潜在的な危険性。

  •  アクセシビリティの要約 — （例えば、拡張された記述の欠如や特定の危険性など）全ての既知な欠陥の説明を格納する、全体的なアクセシビリティの人間が可読可能な要約。

EPUB 出版物は、次のアクセシビリティ メタデータを格納するべきである（SHOULD）。

  •   十分なアクセス モード — 情報を著しく損失することなくコンテンツを消費するために十分な、ひとつ以上のアクセス モードのセット。EPUB 出版物は、格納するコンテンツの種類に応じて、消費するために十分なアクセス モードのセットを複数持つことができる（すなわち、アクセス モードとは異なり、このプロパティは、例えば音声コンテンツのトランスクリプトを格納など、幅広くアクセシブルでないコンテンツのアフォーダンスを考慮に入れている）。

EPUB 制作者は、このセクションで指定されていない追加のアクセシビリティ メタデータを格納してもよい（MAY）。

注

これらプロパティの詳細な情報と EPUB の異なるバージョンに、これらのプロパティを格納する方法は、検出のメタデータの達成方法集 [EPUB-A11Y-TECH-11] を見よ。他のフォーマットにアクセシビリティ メタデータを格納する詳細情報は、DIST-002: 配信レコードにアクセシビリティ メタデータの格納する [EPUB-A11Y-TECH-11] も見よ。

2.3 リンクされたメタデータ レコード

アクセシビリティ メタデータは、リンクされたレコード [EPUB-3] に格納することも可能だが（つまり、link 要素から参照されるメタデータ レコード）、リンクされたレコードのみにそのようなメタデータを格納することは、この仕様書における発見可能性の要求事項を満たさない。

3. アクセシブルな出版物

3.1 導入

このセクションは規定ではない。

EPUB は、コンテンツをオーサリングするために使用されるコア技術の HTML や CSS、JavaScript や SVG といったオープン ウェブ プラットフォーム上に構築する。これらの技術の使用は、EPUB 制作者が、確立されたウェブ アクセシビリティ達成方法集を正しく適用するだけで、容易に高度なアクセシビリティをもって EPUB 出版物の作成ができることを意味する。

アクセシブルなウェブ コンテンツを生み出す一次資料は、W3C の Web Content Accessibility Guidelines (WCAG) [WCAG2] である。この仕様書は、アクセシブルなコンテンツのベンチマークを確立するために、WCAG で成された大規模な作業を活用し、同じ四つのハイレベルのコンテンツの原則（知覚可能、操作可能、理解可能および堅牢性）が、アクセシブルな EPUB 出版物を作成するための中心となっている。

このセクションは、WCAG で定義された適合性基準を適用する方法を定義し、EPUB 出版物特有の性質に取り組む。

このセクションの要求事項に準拠するように制作された EPUB 出版物は、多種多様なニーズと好みを持つユーザーのために高いアクセシビリティを備えているであろう。

3.2 WCAG との関連性

このセクションは規定ではない。

WCAG [WCAG2] 及びその関連した達成方法集は、テーブルから埋め込み型マルチメディア、豊富なセマンティクスに至るまで、ウェブ コンテンツのアクセシビリティに関する問題と解決策の膨大なカバレッジを提供する。これらは、この仕様書を構築する上での基礎を表している。

この仕様書は、（例えば、ガイダンスが同期していなかったり、矛盾しているなど）二つの規格間の互換性を崩すリスクがあるので、これらのドキュメントに導入された要求事項または達成方法集を繰り返さない。同時に、この仕様書は、これらの要求事項を要求していないが、アクセシブルな EPUB 出版物を作成する際にその重要性が低下することはない。

その代わりに、この仕様書は、（単一のページではなくウェブ ドキュメントの集合体としての）EPUB 出版物に WCAG を適用する方法を定義し、追加の要求事項のセットを加える。これらの要求事項は、WCAG で取り上げられているものほど重要ではなく、単純に EPUB 出版物として従うために必要なだけである。（各要件事項は、それぞれのセクションで WCAG との関係を解説する。）

EPUB アクセシビリティ達成方法集ドキュメント [EPUB-A11Y-TECH-11] の達成方法集も同様である。EPUB 出版物に固有又は EPUB 出版物のコンテクストで明確化を必要とする達成方法集のカバレッジを提供する。これは、WCAG 達成方法集の残りの部分は適用されないということを意味しない。

結果として、EPUB 制作者は、WCAG 適合性の深い知識がなくてもこのセクションを読むことができるが、この仕様書のアクセシビリティ要求事項を実装するために、WCAG の理解は要求される。

この仕様書は、WCAG の一部ではない要求事項が追加されているので、EPUB 出版物は、この仕様書に適合することなく WCAG に適合することができる。

3.3 WCAG 適合性

3.3.1 WCAG 適合性要求事項

この仕様書に適合するために、EPUB 出版物は、次の要求事項を満たす必要がある。

  •   WCAG 2.0 [WCAG20] の要求事項を満たさなければならないが（MUST）、最新の推奨バージョンに準拠することが強く推奨される（RECOMMENDED）。

  •   レベル A の要求事項を満たさなければならないが（MUST）、レベル AA [WCAG2] を満たすことが強く推奨される（RECOMMENDED）。

これらの要求事項によって提供されるレポートの柔軟性は、その地域で有効な要求事項を否定したり優先したりすることなく、アクセシビリティが義務づけられている地域でも、この仕様書を適用できることを保証することである。

例えば、この仕様書は、WCAG 2.0 レベル A のベースライン要求事項を設定しているが、主に、古いコンテンツの後方互換性を EPUB 制作者に提供し、正式な要求事項が存在しない場合にアクセシブルな制作物の採用を促すための柔軟性を提供するためである。しかしながら、ほとんどのアクセシビリティ専門家は、このレベルが高度なアクセシビリティを提供しているものとして認識していない。

理想的に、EPUB 制作者はレベル AA の WCAG の最新版に適合するように努めるべきであるが、地域や国内の法律、又は調達先や配信業者の要求事項によって、満たさなければならない正式な閾値が定義される。

注

アクセシビリティに関する法的要求事項の例は、欧州連合の 2019/882 指令やアメリカ合衆国の 1973年リハビリテーション法508条を格納する。EPUB 出版物は、これらの法律に準拠するためには、基本的なレベル A の達成基準以上のものを満たす必要がある。

WCAG に歩調を合わせることは、ユーザーのアクセスを継続的に向上するというメリットがある。ウェブ技術の変化と改善、アクセスを阻害する条件の認識が進化するにつれて、規格は、新しい要求事項を追加する。これらの追加の要求事項を満たすことで、EPUB 出版物が最新の達成方法集を採用していることの保証に役立つ。古いバージョンの要求事項を満たすことは、まだ役立つものの、最適な読書体験を失う結果となる可能性がある。

同様に、法的枠組みやポリシーは、しばしば、アクセシビリティのベンチマークとしてレベル AA の適合性を引用する。その理由は、EPUB 制作者が、現実的に実装できる改善の範囲を最も広く提供するためである（EPUB 制作者は、可能であれば AAA 要求事項を満たすように試みるべきであるが、AAA に完全に適合することは一般的に不可能である）。EPUB 制作者が、レベル A 適合しか満たさないとき、様々なユーザー グループに対するコンテンツに妥協し、最適な読書体験を損なう結果となる。

3.3.2 WCAG 適合性の評価

3.3.2.1 ページと出版物

WCAG 原則 [WCAG2] は、個々のウェブ ページの評価にフォーカスしているが、EPUB 出版物は、WCAG がウェブ ページの集合「共通の目的を共有するウェブ ページの集合」[WCAG2] として言及しているものに類似している。

従って、EPUB 出版物のアクセシビリティを評価するとき、EPUB 制作者は、個別に個々のページ（または EPUB 3 として知られているコンテンツ ドキュメント）を評価できない。むしろ、EPUB 制作者は、より大きな作品の一部として、全体的なアクセシビリティを評価しなければならない（MUST）。

例えば、誤った読み上げ順序でドキュメントをリストアップしている場合、EPUB 制作者が、個々の EPUB コンテンツ ドキュメントに論理的な読み上げ順を与えるだけでは十分ではない。同様に、全ての EPUB コンテンツ ドキュメントにタイトルを格納することは、出版物にタイトルを提供することを補完的するものである。いずれかが欠落している場合、全体的なアクセシビリティが低下する。

EPUB 制作者は、知覚可能、操作可能、理解可能および堅牢性であるコンテンツの WCAG ガイドラインを、それぞれのコンテンツ ドキュメントだけでなく、EPUB 出版物全体に対しての評価しなければならない（MUST）。

EPUB アクセシビリティ達成方法集 [EPUB-A11Y-TECH-11] は、EPUB 出版物にこれらのガイドラインを適用する際の詳細な情報を提供する。

3.3.2.2 適合性基準の適用

EPUB 出版物を評価するとき、WCAG 適合性基準 [WCAG2] は、次のように適用される。

  • 適合性レベルのコンプライアンスを決定するとき、EPUB 出版物全体は、主張されたレベルの適合性要求事項を満たさなければならない（MUST）。
  • EPUB 制作者は、ユーザーがフォールバックにアクセスするための信頼できる方法がないので、適合している代替版 [WCAG2] を提供するために EPUB のフォールバック メカニズムを使用してはならない（MUST NOT）。EPUB 制作者が、フォールバックを使用する場合、重要なコンテンツとそのフォールバックの両方は、主張された適合性レベルの要求事項を満たさなければならない（MUST）。EPUB 固有のフォールバック メカニズムは、マニフェスト フォールバック [EPUB-3] とバインディング [EPUB-3]、epub:switch 要素 [EPUB-3] を介してコンテンツのスイッチを格納する。
  • 「ページ全体」の要求事項 [WCAG2] のコンプライアンスを決定するとき（すなわち、ページの一部は、適合性の表明を作る時に、除くことができない）、各 EPUB コンテンツ ドキュメントの全体は、適合性レベルを達成しなければならなず（MUST）、及び EPUB 出版物内の各コンテンツ ドキュメントは、規定の適合性レベルを満たさなければならない（MUST）。

3.4 EPUB 要求事項

3.4.1 ページ ナビゲーション

3.4.1.1 概要

このセクションは規定ではない。

一般的な読書層と教育環境内の両方の本として最も消費される印刷物のように、静的にページ付けされたコンテンツは、まだ至る所に存在する。印刷物は、静的なページネーションの唯一のソースではなく、静的なページの境界は、固定レイアウトのデジタル出版物にも存在する。

結果として、（例えば、教師が、特定のページをめくるように生徒へ指示するときなど）出版物内の同じ位置を容易に探すことができないため、非視覚の読者は、静的にページ付けされたコンテンツを使用する環境では、同世代の読者と比べて不利な立場に直面する。

ページ境界位置を格納することは、リフロー可能なメディアの選択が、ユーザーの不利にならないことを保証することで、この格差を埋めるのを助ける。

ページ ナビゲーションを提供することは、静的なページ付けと同等のものを持っていないリフロー出版物も助ける。リーディング システムによるこれら出版物のデフォルトのページネーションは、ビューポート サイズやユーザーのフォント設定に応じて変化するので、静的ではない。結果として、同じ EPUB 出版物のユーザー同士で位置を連動させることは、静的な参照なしには困難である。

ページ ナビゲーションの格納は、（例えば、目次、線形な読み上げ順序、その他のナビゲーションの補助に加えてなど）ユーザーがコンテンツにアクセスするために他の有意義な方法を提供することで、複数の手段の達成基準 [WCAG2] を達成するためのひとつの方法を表している。

印刷物/デジタルが混在する環境におけるページ ナビゲーションの重要性を考慮すると、この機能を格納するという要件は、複数の手段の達成基準を満たすための多くの方法のひとつとしてよりも高い優先順位を持つ。

注

EPUB 出版物にページ ナビゲーションを格納するについての詳細な情報は、ページ マーカー [EPUB-A11Y-TECH-11] を見よ。

3.4.1.2 適用性

EPUB 出版物は、次の例のいずれかに該当する場合、このセクションで定義されたページ ナビゲーションの目的を格納するべきである（SHOULD）。

  • EPUB 制作者が、（例えば、印刷物/デジタルをセットに格納された）静的にページ付けされた出版物の動的にページ付けされた版として EPUB 出版物を識別し、
  • EPUB 制作者が、（例えば、教育現場など）両方のバージョンを使用することが合理的に予測できる環境において、静的にページ付けされた出版物の代替として EPUB 出版物を提供する、又は
  • EPUB 制作者が、フォーマット間の改ページ位置の保持を可能にするワークフローから EPUB 出版物と静的にページ付けされた出版物を生成する。

EPUB 制作者は、静的にページ付けされた同等ではないリフロー可能な EPUB 出版物にページ ナビゲーションを格納してもよい（MAY）。

3.4.1.3 目的
3.4.1.3.1 ページネーションのソース
目的

静的な改ページの位置のソースを特定する。

この目的を理解する

ユーザーは、自分のニーズに合うかどうかを判断するために、EPUB 出版物のページネーションのソースを知る必要がある。例えば、ハードカバー版とソフトカバー版の両方で製作された印刷物は、ページネーションが異なるだろう。また、同じ本の異なる版でも、ページネーションが異なる場合がある。

例えば、ISBN 又は ISSN など、静的にページ付けされたソースに認識可能な識別子を格納することで、ユーザーはページネーションがどのバージョンに対応しているかを確実に判断できる。

EPUB 制作者が、デジタルのみの出版物のナビゲーション補助としてページ付けを挿入する場合、ソースを指定してはならない（すなわち、現在の出版物を自身のページネーションの出典として特定してはならない）。

この目的を満たす

EPUB 出版物が、出版物の静的なページ付けバージョンに対応する改ページ マーカー及び/又はページ リストを格納するとき、EPUB 制作者は、パッケージ ドキュメント メタデータでそのソースを特定しなければならない（MUST）。

3.4.1.3.2 ページ リスト
目的

静的な改ページ位置へのナビゲーションを提供する。

この目的を理解する

ページ リストは、EPUB 出版物の静的な改ページ位置へのリンクのリストを提供するので、改ページ位置へ移動するための主要な手段となる。

リーディング システムは、通常、ユーザーが移動したいページ番号を入力できる「ページ移動」インターフェースを生成するためにこのリストを使用するが、ユーザーが、全てのリストにアクセスして移動するページ番号を選択する機能を提供する場合もある。

ページ リストがないと、これは、個々の改ページ マーカー（もし存在する場合）のナビゲーションに頼るので、ページ ナビゲーションが非常に困難になる。

この目的を満たす

EPUB 出版物は、ページ リストを格納しなければならない（MUST）。

EPUB 制作者は、ソース（すなわち、デジタル版で再現されていない空白のページ又はコンテンツへのリンクを提供する必要はない）から再現されたコンテンツの全ページへのリンクを格納するべきである（SHOULD）。

EPUB 制作者は、再現されているかどうかに関わらず、ソース内の全ページへのリンクを格納するべきであるが、これは要求事項ではない。

3.4.1.3.3 改ページ
目的

静的な改ページ位置を提供する。

この目的を理解する

EPUB 出版物に改ページマーカーを挿入すると、テキスト内のどこにいるかに関するコンテキストをユーザーに提供される。支援技術は、例えば、ユーザーがページ上の何かを引用したい場合など、ユーザーが現在いるページ番号を通知するためにこの情報を使用できる。

改ページ マーカーを格納することで、ユーザーは毎回、ページ リストにアクセスすることなく、ページごとに素早く前後に移動することもできる。

これらマーカーの格納は、リンク先を簡単に参照できるため、ページ リストの作成も容易になる。

この目的を満たす

EPUB 出版物に改ページ マーカーを格納することは、オプションである（OPTIONAL）。

EPUB 制作者が、改ページ マーカーを格納する場合、

  • ソースから再現された全ページに改ページ マーカーを格納するべきである（SHOULD）（すなわち、デジタル版で再現されていない空白ページ及びコンテンツに、マーカーは必要ない）。
  • （再現されているかどうかに関わらず）ソース内の全ページに改ページ マーカーを格納するべきであるが、これは要求事項ではない。

加えて、（例えば、EPUB 3 メディア オーバーレイ [EPUB-3] など）コンテンツのテキストと音声の同期再生でページ番号が読み上げられる場合、EPUB 制作者は、再生を制御するマークアップでページ番号を特定しなければならない（MUST）。

3.4.2 メディア オーバーレイの再生

3.4.2.1 概要

このセクションは規定ではない。

メディア オーバーレイは、テキストと音声が同期していることから利益を得られる全ての人に、アクセシブルな再生体験を提供する。これらはまた、音声再生のみの要求であったり、テキストのハイライトを伴う読書の利益のみを必要とするユーザーに役立つ。メディア オーバーレイはまた、全てのユーザーのために EPUB 出版物の始めから終わりにかけてシームレスな再生体験を可能にする。

しかしながら、最も基本的なメディア オーバーレイ ドキュメント [EPUB-3] は、リーディング システムに最小の構造のみを提供する。これらは、ハイライトするテキストと、テキストに対応するオーディオ クリップを示している。結果的に、ユーザーは、基本的な再生と停止のオプションのみを利用できる。

EPUB 制作者は、リーディング システムがより使いやすい体験を提供できるように、メディア オーバーレイ ドキュメントに構造とセマンティクスを追加する必要がある。リッチなマークアップがあれば、リーディング システムは、主要な物語に干渉する二次的なコンテンツを飛び越える能力や、表のように深くネストした構造からユーザーを脱出させたり、目次に行くことなく出版物のセクションを移動する能力を提供できる。

メディア オーバーレイ ドキュメントに構造と意味を加えることは、情報及び関係性の達成基準 [WCAG2] の目的に広く該当する。構造化及び意味のある再生シーケンスがなければ、コンテンツの豊かなナビゲーションをユーザーから奪うことになる。

3.4.2.2 適用性

メディア オーバーレイ ドキュメントは、[EPUB-3] の要求事項を満たさなければならない（MUST）。この仕様書に適合するために、[EPUB-3] で定義されている以上の追加の要求事項を満たす必要はない。

しかしながら、様々な読書ニーズを持つ人々に対してメディア オーバーレイの効果を最大にするため、EPUB 制作者は、次のセクションで定義されている省略可能な目標を満たすことが強く推奨される。

注

EPUB 制作者は、EPUB 出版物にメディア オーバーレイを格納する必要はないが、存在する時、これらの要求事項への適合を確認する必要がある。

3.4.2.3 目的
3.4.2.3.1 完全性
目的

全文が音声で聴けるように確認する。

この目的を理解する

音声形式の出版物を必要とするユーザーは、音声合成による再生を利用することができるが、録音済みのナレーションが提供されているときに比べて、かなり劣った体験となる。音声合成エンジンは、内蔵する語彙が限られているため、遭遇するほとんどの珍しい単語は、そのほとんどは支離滅裂であり、誤って発音される。その結果、ユーザーはコンテンツの意味を理解するために、言葉を繰り返したり、綴ったりしなければならず、読むスピードが遅くなり、理解力が低下する。

このような理由から、全テキストに加えて、出版物の全テキストにナレーションを提供することが重要である。ユーザーは、テキスト、音声又はその両方を組み合わせて、どちらの読み方を好むか決定できる。

この目的を満たす

EPUB 制作者は、全てのテキスト コンテンツに対して、メディア オーバーレイを介して同期音声の再生を提供しなければならない（MUST）。

3.4.2.3.2 読み上げ順序
目的

メディア オーバーレイの再生が、論理的な読み上げ順序と一致していることを確認する。

この目的を理解する

全ての EPUB 出版物は、ユーザーがコンテンツを介して論理的に進めることができるように、デフォルトの読み上げ順序を持つ。読者が主要なストーリーを追うことができ、最も意味のあるところで副次的なコンテンツに出会うことができます。デフォルトの読み上げ順序は、セルからセル、行から行へのテーブル内を進むように、いくつかの明白でない関係も確立する。

メディア オーバーレイ ドキュメントの par と seq 要素のシーケンスが、この連続性と一致しない場合、読者が音声のみを聞いているのか、視覚的にもフォローしようとしているのかに関わらず、読者の混乱を引き起こす可能性がある。

デフォルトの読み上げ順序に一致するように再生を順序付けることは、ユーザーがテキストを辿ることが確実できるようにする最も安全な方法である。しかしながら、場合によっては、この方法に厳密に従うと、（例えば、行ではなく列ごとに表を再生するほうが、より論理的な意味になる場合があるなど）読書体験が最適化されない可能性がある。この目標の目的は、代替のプレゼンテーションを禁止することではなく、逸脱することがコンテンツの論理的な流れを維持することを確実にすることである。

この目的を満たす

EPUB 制作者は、次の両方を反映するように、メディア オーバーレイ ドキュメントの par と seq 要素を並べるべきである（SHOULD）。

  • spine [EPUB-3] 内の参照された EPUB コンテンツ ドキュメントの順序、そして、
  • 各 EPUB コンテンツ ドキュメント内の各要素の順序。

EPUB 制作者が、異なる順序を使用する場合、その順序は、コンテンツの論理的な再生をもたらさなければならない（MUST）。

3.4.2.3.3 スキップ可能性
目的

ユーザーが自動的にコンテンツをスキップできるようにする。

この目的を理解する

作品の主要な物語を中断することなく読むことができることは、読解力の中心である。EPUB 制作者は、典型的に、主な物語の流れから外れている、例えば、補足及び脚注などの副次的な情報を、（例えば、様々な背景色や配置を使用することで、読者が、読書中にこの情報へ視覚的にフィルターをかけることができるなど）視覚的に表現するために EPUB 出版物を構成する。

しかしながら、耳で読むことを好む読者は、デフォルトでは容易にこのような情報をスキップすることができない。メディア オーバーレイの再生は、典型的に、リーディング システムが、副次的なコンテンツが発生する場所でレンダリングする。

しかしながら、EPUB 制作者が、メディア オーバーレイ ドキュメントに構造的なセマンティクスを追加するとき、リーディング システムは、ユーザーが、再生時にデフォルトでスキップする副次的なコンテンツを決定できるような読書体験を作ることができる。

この目的を満たす

EPUB 制作者は、メディア オーバーレイ ドキュメント内の全てのスキップ可能な構造 [EPUB-3] を特定するべきである。

3.4.2.3.4 回避可能性
目的

構造化されたコンテンツからユーザーが自動的に回避できるようにする。

この目的を理解する

視覚的に読書するとき、ユーザーは、例えば、表、リスト、及び図などの高度に構造化されたコンテンツの中を素早く移動し、そこから回避できる。例えば、行と列を使用する表のレイアウトは、ユーザーはどちらかの軸に沿って素早く必要な情報を見つけ、終わったら簡単に主要な物語に戻ることができる。同様に、項目のリストはざっと見て、必要な情報を見つけたらそこから回避できる。

コンテンツから回避する同様の容易さは、デフォルトではメディア オーバーレイ ドキュメントにおいては利用できない。EPUB 制作者が、ドキュメントにこれらの要素の構造的なセマンティクスをエンコードしない限り、ユーザーはテーブルのセル、行、又はテーブル自体から回避できない。

EPUB 制作者が、この情報を提供するとき、リーディング システムは、同等の読書体験ができるように、聴覚による読者のための再生を簡素化できる。

この目的を満たす

EPUB 制作者は、メディア オーバーレイ ドキュメント内の全ての回避可能な構造 [EPUB-3] を特定するべきである。

3.4.2.3.5 ナビゲーション ドキュメント
目的

リーディン システムによって提示されたとき、EPUB ナビゲーション ドキュメントに役立つナビゲーションが、聴覚による再生が可能であることを確認する。

この目的を理解する

通常、リーディング システムは、EPUB ナビゲーション ドキュメントに役立つナビゲーションに独自のインターフェースを提供する。例えば、ユーザーが読んでいるコンテンツの上部に、専用のインターフェースとして目次を開くなど。

これらのインターフェーイスにアクセスするために、通常、利用可能なとき、ユーザーは、エントリを聞くために、音声合成の再生する必要がある。

EPUB ナビゲーション ドキュメントにメディア オーバーレイ ドキュメントを提供することで、リーディング システムは、リンクに聴覚的なラベルを使用することができ、聴覚による読者の体験を向上させる。

この目的を満たす

EPUB 制作者は、EPUB ナビゲーション ドキュメント [EPUB-3] にメディア オーバーレイ ドキュメントを格納するべきである。

3.5 適合性をレポートする

3.5.1 導入

このセクションは規定ではない。

評価者は、EPUB パッケージ ドキュメント内にあるメタデータ プロパティの表現を介して、EPUB 出版物のアクセシビリティ適合性を報告する。

このメタデータは、次の両方を確立する。

  • 達成された適合性のレベル、そして、
  • 評価を行った人に関する情報。

メタデータは、次のセクションで詳細に説明されているように、DCMI メタデータ用語 [DCTERMS] と EPUB アクセシビリティ語彙からプロパティの組み合わせを使用する。

3.5.2 出版物の適合性

EPUB 出版物が、この仕様書のアクセシビリティ要求事項に適合していることを示すために、その値が、次のパターン（すなわち、大文字及び小文字と間隔の両方に）正確に一致しなければならず（MUST）、conformsTo プロパティを格納しなければならない（MUST）。

EPUB-A11Y-A11Y-VER_WCAG-WCAG-VER-WCAG-LVL

ここでは、

A11Y-VER

EPUB 制作者は、小数点以下を格納せずに EPUB アクセシビリティ仕様のバージョン番号を指定しなければならない（MUST）。EPUB 制作者は、この仕様書のこのバージョンを示すために値 11 使用しなければならない（MUST）。

WCAG-VER

EPUB 制作者は、（例えば、WVAG 2.0 は 2.0 又は WCAG 2.1 は 2.1 など）小数点以下を格納せずに出版物が適合する WCAG のバージョン番号を指定しなければならない（MUST）。

WCAG-LVL

EPUB 制作者は、（例えば、A 又は AA など）出版物が適合する WCAG 適合性レベルを指定しなければならない（MUST）。

次の適合の文字列は、この仕様書の発行時点で有効である。

  • EPUB-A11Y-11_WCAG-20-A
  • EPUB-A11Y-11_WCAG-20-AA
  • EPUB-A11Y-11_WCAG-20-AAA
  • EPUB-A11Y-11_WCAG-21-A
  • EPUB-A11Y-11_WCAG-21-AA
  • EPUB-A11Y-11_WCAG-21-AAA
注

妥当な適合性文字列のリストは、W3C が WCAG の新バージョンをリリースするたびに増加する。

例 1

次の例は、WCAG 2.1 レベル AA の EPUB アクセシビリティ 1.1 仕様に適合する EPUB 3 出版物の適合性ステートメントを示している。

<package …>
	   <metadata>
	      …
	      <meta property="dcterms:conformsTo">EPUB-A11Y-11_WCAG-21-AA</meta>
		  …
	   </metadata>
	   …
	</package>
注

将来の WCAG の 3.0 バージョンでは、新しいレベル名 (現在は、ブロンズ、シルバー、及びゴールド) も導入されるようである。これらの名前は、文字列パターンの A、AA、及び AAA を置き換えるものである。

注

WCAG 適合性要求事項のみを満たしている（つまり、この仕様に完全に適合していない）EPUB 出版物は、confomsTo プロパティに WCAG 適合性 URL（例えば、[WCAG21] の「https://www.w3.org/TR/WCAG21/」など）を使用できる。

3.5.4 適合性の再評価

このセクションは規定ではない。

注

次のガイダンスは、EPUB 制作者が、新しい評価が必要な時を判断するのに役立つ。この仕様書に適合することを要求されるものではない。

EPUB 出版物の適合性評価がどのくらいの期間有効であるかは、複雑な問題である。継続的に進化するウェブ サイトとは異なり、EPUB 制作者は、初版後に EPUB 出版物を更新することはないだろう。その結果、変更されていない EPUB 出版物は、常にその最後の評価に適合する。

しかしながら、出版業界は、作品の間違いや誤植を修正するために EPUB 出版物の重版をリリースしたり、定期的に新しい版をリリースしたりするのが一般的である。EPUB 出版物への全ての変更が、そのアクセシビリティを実質的に変更するわけではないため、EPUB 制作者が、完全又は部分的な再評価が十分なのかだけでなく、新しい評価をいつ行うべきかも問題は複雑になる。

ルールとして、EPUB 制作者は、EPUB 出版物の構造と機能に実質的な変更を加えたときはいつでも、コンテンツを再評価しなければならない。例えば、

  • EPUB コンテンツ ドキュメントのマークアップの性質又は順序の変更、
  • 情報を伝える画像の追加又は変更、
  • （例えば、コントラストなど）可読性に影響を与えるフォーマットへの変更、そして、
  • インタラクティブなコントロール、フォーム、その他への追加又は変更。

EPUB 出版物が以前のリリースと実質的に同じマークアップ及びコンテンツを格納する場合、EPUB 制作者は、適合性を再確認するために新しい変更を評価するだけでよい。

しかしながら、EPUB 出版物の最後のリリースから、この仕様書又は [WCAG] の更新版が発行された場合、この仕様書は、最新の規格への適合性を確認するために、新しい評価を実行することも推奨する。EPUB 制作者は、この場合でも完全な再評価を実行する必要ないかもしれない（つまり、規格が方法や適合性に大きな変更を加えない限り、新しい又は変更された達成基準のみをチェックのみする必要があるかもしれない）。

逆に言うと、EPUB 制作者は、以下のような実質的でない変更を行うとき、再評価を実行する必要はない。例えば、

  • テキストの誤字脱字の修正（つまり、マークアップの変更はない）、
  • 装飾画像の追加又は変更、
  • コンテンツの理解に影響しない又はテキストの表示を変更しないフォーマットへの変更、及び、
  • パッケージ ドキュメント メタデータの変更。

EPUB 出版物のアクセシビリティを評価する資格のある個人は、変更が実質的なものであるかどうかを判断するべきである。例えば、編集者は、見たところ小さなフォーマット変更の影響を認識していない可能性がある。

実質的でない変更の場合でも、アクセシビリティの標準が変更された場合、この仕様書は評価の更新（全体又は部分的な）を推奨する。

EPUB 制作者は、ここで説明されたものよりもさらに進歩的なアプローチを考慮するべきである。EPUB 出版物のアクセシビリティを再評価及び更新する前にコンテンツの変更を待っていると、最新の改善点が不足する可能性がある。例えば、出版社は、可能な限り幅広いオーディエンスが最大限に利用できることを維持するために、ベストセラーの EPUB 出版物の定期的なレビューを優先するかもしれない。

4. 最適化された出版物

WCAG [WCAG2] は、コンテンツを広くアクセシブルにするための一般的なガイドラインのセットを提供するが、適合するコンテンツが必ずしも特定のユーザー グループにとって最適であるとは限らない。逆に、特定のニーズや読書モダリティに最適化されたコンテンツは、特定のオーディエンスを対象としているので、WCAG へ正確に適合していないことが多い。

例えば、テキストと音声を同期させた EPUB 出版物は、コンテンツの完全な音声録音を含むかもしれないが、主要な見出しのみにテキスト コンテンツを限定できる。この場合、EPUB 出版物は、コンテンツを聴く必要のある（すなわち、完全な出版物を聴くことができ、見出しの間を移動することができる）ユーザーによって消費可能であるが、音声を聴くことができない人にとっては使用できない。

言い換えれば、EPUB 制作者が、特定の読書様式に対して EPUB 出版物を最適化する場合、WCAG 適合性レベルを達成しなかったとしても、意図された読者がアクセスしにくくなるわけではない。

EPUB 制作者は、[DCTERMS] に従い conformsTo プロパティによって、最適化された EPUB 出版物が準拠している規格またはガイドラインを特定しなければならない（MUST）。その値として、EPUB 制作者は、コンテンツが従う標準又はガイドラインを参照する [URL] に従う URL を指定しなければならない（MUST）。

例 9

次の例は、DAISY Navigable Audio-only EPUB 3 Guidelines [DAISYAudio] に準拠した EPUB 3 出版物の適合性のステートメントを示している。

<package …>
	   <metadata>
	      …
	      <link rel="dcterms:conformsTo" href="http://www.daisy.org/guidelines/epub/navigable-audio-only-epub3-guidelines"/>
		  …
	   </metadata>
	   …
	</package>

URL が、（例えば、ガイドラインが公開されていないなど）適合性を理解するのにユーザーにとって十分でない場合、コンテンツが最適化されている方法に関する詳細情報を、アクセシビリティの要約で提供されるべきである（SHOULD）。

例 10

次の例は、点字のレンダリングに最適化された EPUB 出版物のアクセシビリティの要約を示している。

<meta property="schema:accessibilitySummary">
		This publication is optimized for braille readers. It will not be 
		usable by persons who cannot read braille. The publication is designed
		for braille reading devices capable of displaying 6-character cells and
		40-character line lengths. The text is not contracted, and follows 
		Unified English Braille formatting conventions. All characters are
		encoded using the Unicode braille character set.
	</meta>
例 11

次の例は、音声のレンダリングに最適化された EPUB 出版物のアクセシビリティの要約を示している。

<meta property="schema:accessibilitySummary">
		This publication is an audio book. It will not be usable by persons who 
		cannot hear the audio. The publication is recorded by a professional
		narrator. There is navigation to the beginning of each chapter. The text
		of the publication is not included. Images are not included, but the
		photo captions are narrated at the end of the chapter where they occur.
	</meta>
注

この仕様書は、最適化されたコンテンツを制作するための規格及びガイドラインの定義又は推奨をしない。しかしながら、このような標準の情報リストは、EPUB Optimized Publication Standards Registry を参照されたい。

5. 配信

このセクションは規定ではない。

注

EPUB作成者は、この仕様書に適合するためにこのセクションの推奨事項に従う必要はないが、一部の法域は EPUB 制作者が同様の慣行に従うことを要求しており、例えば、2019/882 指令は、欧州連合内で配信されるデジタル出版物に対する同様の要求事項が含まれている。

アクセシブルな EPUB 出版物の制作は、その出版物が、アクセシブルな方法でユーザーによって入手可能または消費可能であることを保証するものではない。EPUB 制作者は、これらの EPUB 出版物を配布する方法に応じて、他の要因が、これらの全体的なアクセシビリティに影響するだろう。例えば、アクセシビリティ メタデータの検索とレビューをする機能と同様に、コンテンツを検索と入手のためのアクセシブルなインターフェースは、配布プロセスの重要な部分である。

配信プロセスの多くは、EPUB 制作者のコントロール外とはいえ、この仕様の範囲外であるが、EPUB 制作者がコントロールできる要素もある。例えば、EPUB 制作者は通常、その EPUB 出版物に適用されるデジタル著作権管理（DRM）スキームのアクセシビリティをコントロールしないが、その EPUB 出版物に適用する使用権をコントロールする。従って、DRM スキームによって著者が出版物のテキストへのアクセスをブロックすることができても、EPUB 制作者は、テキストを読み上げる支援技術の機能をブロックする可能性があるため、そのような制限を適用しないように注意する必要がある。

配布によるアクセシビリティへの影響を最小限に抑えるため、この仕様書は、次の配布方法を忠実に実行することを EPUB 制作者に勧める。

  • 支援技術によるアクセスを阻害するような制限を課す必要はない。そして、
  • フォーマットがそのようなメタデータをサポートしているとき、（例えば [ONIX] 又は [MARC21] など）EPUB 出版物の配信に必要なレコード フォーマットに、アクセシビリティ メタデータを格納する必要がある。
注

配信業者は、EPUB 制作者の瑕疵なしにアクセシビリティを本質的に損なうようなデジタル著作権管理スキームを実装することができる。このセクションのガイダンスに従うことは、EPUB 制作者が、そのような配信業者の使用を制限することではない。この意図は、EPUB 制作者が通常ではアクティブではない機能をアクティブにすることで、アクセシビリティを損なわないことだけである。

編集者注

A. EPUB アクセシビリティ語彙

A.1 概要

A.1.1 この語彙について

この語彙は、パッケージ ドキュメントのメタデータにおける EPUB 出版物のアクセシビリティを記述するためのプロパティを定義する。

A.1.2 参照

この語彙を参照するためのベース URL は、http://www.idpf.org/epub/vocab/package/a11y/# である。

この仕様書は、この語彙のプロパティで使用するためにプレフィックス「a11y:」を予約する。EPUB 作成者は、パッケージ ドキュメントでプレフィックスを宣言する必要はない。

A.1.3 適合性プロパティ

A.1.3.1 certifiedBy
名前： certifiedBy
概要： EPUB 出版物のアクセシビリティのテスト及び認証に責任を持つ当事者を特定する。
許可された値： xsd:string
個数： ひとつ以上
例： 
<meta property="a11y:certifiedBy">Accessibility Testers Group</meta>
A.1.3.2 certifierCredential
名前： certifierCredential
概要： アクセシブルなコンテンツを認証するために、関連する certifiedBy プロパティで識別された当事者の権限を確立する資格又はバッジを識別する。
許可された値： xsd:string
個数： 0個又は複数
拡張できる対象： a11y:certifiedBy
例： 
<meta property="a11y:certifiedBy" id="certifier">Accessibility Testers Group</meta>
	<meta property="a11y:certifierCredential" refines="#certifier">DAISY OK</meta>
A.1.3.3 certifierReport
名前： certifierReport
概要： 関連する certifiedBy プロパティで識別された当事者によって作成されたアクセシビリティ レポートへのリンクを提供する。
許可された値： xsd:anyURI
個数： 0個又は複数
拡張できる対象： certifiedBy
例： 
<meta property="a11y:certifiedBy" id="certifier">Accessibility Testers Group</meta>
	<link rel="a11y:certifierReport" refines="#certifier" href="http://example.com/a11y/reports/9780000000001"/>

B. 更新履歴

このセクションは規定ではない。

この変更履歴は、EPUB 出版物の適合性に影響を与える又は同様に注目すべき、EPUB Accessibility 1.0 以降の大きな変更のみを識別することに注意されたい。

改訂の間に取り上げられた全ての問題のリストについては、ワーキング グループのイシュー トラッカーを参照されたい。

  • 2021年6月9日：改ページ マーカー及び/又はページ リストが、デジタルのみの出版物に格納されているとき、ページネーションのソースを指定してはならないことを明確にした。イシュー 1599 を見よ。
  • 2021年4月29日：平文の文字列と混同されないようにするために、適合性識別子にハイフンやダッシュを使用するように変更。イシュー 1455 を見よ。
  • 2021年3月26日：EPUB 出版物をいつ再評価するかについての規定のセクションを追加。イシュー 1470 を見よ。
  • 2021年3月12日：配信セクションを規定に変更したが、（例えば EU など）法律で要求されている場合は要求事項に従わなければならない注記を追加した。イシュー 1487 を見よ。
  • 2021年3月8日：論理的な読み上げ順序を反映させるために、メディア オーバーレイ ドキュメントの par 及び seq 要素のシーケンスのオブジェクトを追加。イシュー 1556 を見よ。
  • 2021年3月5日：ソースから複製されたコンテンツの全てのページに対してページ マーカーを格納することを推奨し、ソース内の全てのページに対してマーカーを格納することをベスト プラクティスとして追加した。イシュー 1502 を見よ。
  • 2021年3月5日：ページ リストを、ソースから複製されたコンテンツの全てのページに対してリンクを格納することを推奨し、ソース内の全てのページに対してリンクを格納することをベスト プラクティスとして追加した。イシュー 1503 を見よ。
  • 2021年3月5日：ページ ナビゲーションとメディア オーバーレイの見出しの下にまとめられていた個々の目的を分割し、EPUB 要求事項セクションを再構成した。。イシュー 1458 を見よ。
  • 2021年2月25日：1.0 仕様への適合性を報告するために使用される IDPF の URL を、より柔軟なテキスト文字列に置き換えた。イシュー 1455 を見よ。
  • 2021年2月19日：WCAG 2.0 への参照は、WCAG 2 への日付のない参照に更新された（WCAG 2.0 が適合性について明示的に言及されている場合を除く）。
  • 2021年2月1日：メディア オーバーレイが EPUB 仕様の要求事項に適合するという勧告を、要求事項に変更した。イシュー 1486 を見よ。
  • 2020年11月17日：certifierCredential 及び certifierReport プロパティと例に refines 属性を追加した。イシュー 1410 を参照されたい。
  • 2020年11月16日：オプションのアクセシビリティ制御および API メタデータへの言及は、発見性セクションから削除されました。これらのメタデータ プロパティは、リーディング システムへ追加で適用可能である。イシュー 1327 を見よ。
  • 2020年9月26日：アクセシビリティ 1.0 仕様を ISO 規格として発行する際に行った改訂を、初稿に反映させた。
